# database-testing
SQL-based QA project simulating banking database testing, covering data integrity, business rules validation, fraud detection, and relational database testing.

PROJECT OVERVIEW This project demonstrates database testing skills using a banking dataset from Kaggle. It is designed to simulate real-world QA scenarios in a financial system, focusing on data integrity, business rules validation, fraud detection, and relational database testing. The goal is to showcase how a QA Engineer validates backend data using SQL, ensuring accuracy, consistency, and reliability across a banking system.

PROJECT OBJECTIVES -Validate data integrity within a flat transactional dataset -Test business rules applied to banking transactions -Detect fraud and suspicious activities -Implement relational database structure

DATABASE DESIGN The project evolves from a flat dataset into a relational database model with the following tables: Transactions, Accounts, and Merchants Tables

TESTING APPROACH: Data Integrity Testing, Business Rule Testing, Fraud Detection Testing, Relational Integrity Testing

TOOLS & TECHNOLOGIES SQL (MySQL) SQL Workbench GitHub (for version control and portfolio showcase)

KEY QA SKILS DEMONSTRATED -Writing quality SQL queries -Database validation techniques -Understanding of relational database concepts -Analytical thinking for fraud detection -Structuring test cases for backend systems
